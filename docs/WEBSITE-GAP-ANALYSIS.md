## 第一轮：PRD模块对照

### 模块A：入学与接入（Enrollment）

| 维度 | 分析 |
|------|------|
| **PRD要求** | CLI接入（`lobster enroll`）、Web Console上传/连接、API批量导入；自动识别Agent信息（名称/框架/版本/模型/工具/工作流）；生成Agent指纹；接入前权限说明；在线/离线模式；10分钟完成接入；最小权限默认 |
| **当前有什么** | ✅ `/check`页有CLI安装指引；✅ `/api/enroll` API存在；✅ 首页有CLI演示 |
| **缺什么** | ❌ **没有Web Console入学向导**（向导式UI引导用户完成接入）；❌ **没有Agent信息自动识别展示页**（框架/模型/工具清单的可视化确认）；❌ **没有权限边界说明页**（接入前告知用户数据采集范围）；❌ **没有离线模式说明/入口**；❌ **没有API批量导入文档**；❌ **没有Agent指纹生成确认流程**；❌ **没有接入进度追踪**（当前状态/下一步引导） |

### 模块B：龙虾护照（Lobster Passport）

| 维度 | 分析 |
|------|------|
| **PRD要求** | 完整护照字段（agent_id, owner_id, framework, model_hash, config_hash, tool_manifest_hash, skill_manifest_hash等）；7状态机（Draft→Under Review→Active→Warning→Restricted→Revoked→Expired）；变更策略（新增技能→Warning，核心变更→Restricted）；证书绑定到个体 |
| **当前有什么** | ✅ `/passport`页显示基础信息、评分、心跳、变更历史 |
| **缺什么** | ❌ **护照字段不完整**（缺owner_id, model_hash, config_hash, tool_manifest_hash, skill_manifest_hash）；❌ **没有完整状态机可视化**（当前只有status文本，没有状态流转图）；❌ **没有变更策略说明**（用户不知道什么操作会导致什么状态变化）；❌ **没有签发者签名/有效期显示**；❌ **没有护照版本历史**（多次认证的记录）；❌ **没有护照状态变更通知设置** |

### 模块C：评测与认证（Academy Cert）

| 维度 | 分析 |
|------|------|
| **PRD要求** | L1/L2/L3能力等级；6种专业标签（DevOps/Knowledge/Customer Service/Security/Software Engineering/Compliance）；S1/S2/S3安全等级；6种评测类型（静态/动态/故障注入/对抗/实战/突击）；9个评分维度；防作弊机制；考试锁定/哈希校验 |
| **当前有什么** | ✅ `/check`页有模拟评分（安全/推理/工具/合规/可靠性5维度）；✅ `/courses`页有4大学院课程体系；✅ `/verify`页有证书验证 |
| **缺什么** | ❌ **没有认证等级展示页**（L1/L2/L3 + S1/S2/S3 + 专业标签的完整体系说明）；❌ **没有考试页面**（实时考试/答题界面）；❌ **没有详细的9维度评分展示**；❌ **没有防作弊机制说明**；❌ **没有认证申请流程页**（选择专业方向→预约考试→参加考试→获取证书）；❌ **没有证书详情页**（显示适用范围、有效期、变更规则）；❌ **没有突击复核机制**；❌ **没有考试历史记录** |

### 模块D：持续监控与复测（Academy Guard）

| 维度 | 分析 |
|------|------|
| **PRD要求** | 定期心跳评测；记录成功率/时延/错误类型/权限异常/工具异常；检测配置/模型/工具/技能变化；漂移预警；局部复测；增量认证；证书自动降级；触发器（工具变更/技能变更/工作流变更/模型变更/连续3次下滑/安全探针异常/权限异常增加） |
| **当前有什么** | ✅ `/monitoring`页有心跳历史表格；✅ `/api/heartbeat` API存在 |
| **缺什么** | ❌ **没有漂移预警仪表板**（实时告警、预警级别）；❌ **没有触发器配置界面**（自定义告警规则）；❌ **没有局部复测入口**（检测到问题后一键触发复测）；❌ **没有增量认证流程**；❌ **没有证书降级通知**；❌ **没有详细错误类型/权限异常/工具异常的分类展示**；❌ **没有Webhook/邮件告警集成**；❌ **没有监控趋势图表**（成功率/时延趋势线）；❌ **没有多Agent统一监控面板** |

### 模块E：技能市场（Academy Exchange）

| 维度 | 分析 |
|------|------|
| **PRD要求** | 上架对象（Prompt Skeleton/Workflow Template/Tool Adapter/Skill Pack/Memory Policy/Adapter-LoRA）；3层市场（社区区/认证区/企业区）；签名验证；沙箱安装；权限声明；审计摘要；变更日志；更新触发证书复核 |
| **当前有什么** | ✅ `/skills`页有技能列表、分类、安装/卸载、价格 |
| **缺什么** | ❌ **没有市场分层**（社区区/认证区/企业区）；❌ **没有技能上架流程**（开发者上传/签名/审核/发布）；❌ **没有技能详情页**（完整描述/权限声明/审计报告/变更日志/评价）；❌ **没有签名验证可视化**；❌ **没有开发者仪表板**（管理自己发布的技能）；❌ **没有技能评价/评论系统**；❌ **没有技能版本管理**；❌ **没有安全审计报告展示**（企业区必备）；❌ **没有技能包更新对证书影响的提示** |

## 第五轮：优先级排序

### 🔴 P0：没有就不能用（MVP必备）

| # | 缺失功能 | 对应模块 | 说明 |

| 7 | **证书详情页** | 模块C | 当前证书验证只显示基本信息，缺有效期/适用范围/签发者 |
