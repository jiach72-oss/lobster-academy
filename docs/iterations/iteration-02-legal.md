# 龙虾学院 · 第2轮迭代：刑部·法务风险深度评审

> 迭代时间：2026-03-24 00:02
> 迭代角度：法律合规

## 核心发现

### 1. "认证" vs "评测" 的法律红线

**中国的认证制度：**
- 根据《认证认可条例》，"认证"是法定资质，需CNCA批准
- 未经批准使用"认证"字样，最高罚20万
- 龙虾学院**绝对不能**自称"认证机构"

**合规表述方案：**
| ❌ 禁止 | ✅ 替代 |
|---------|---------|
| Agent认证 | Agent能力评测/评估 |
| 认证证书 | 评测报告/能力证明 |
| 认证机构 | 第三方评估服务商 |
| 通过认证 | 通过评测 |
| 认证标准 | 评测标准/评测框架 |

**核心逻辑：** 龙虾学院卖的是"评测报告"不是"认证证书"。评测报告是技术服务合同的产物，不需要CNCA资质。认证证书是行政许可的产物，需要CNCA批准。

### 2. 责任边界设计

**风险场景：** Agent通过龙虾学院评测后犯错（如泄露数据、错误决策），客户追责。

**合同设计要点：**
1. **免责条款：** "本评测报告仅反映评测时刻的Agent能力状态，不对Agent后续运行中的行为承担任何责任"
2. **责任上限：** 合同总金额为限（不承担连带责任）
3. **告知义务：** 明确告知评测的局限性（不是100%保证）
4. **举证责任：** 客户需证明评测方法有缺陷，而非Agent本身行为有问题

**类比体检：** 医院体检报告说你健康，之后你生病了，你不能找医院赔。同理，龙虾学院评测说Agent合格，之后Agent犯错，不等于学院赔。

### 3. 数据隐私合规

**PIPL（中国个人信息保护法）关键要求：**
- 评测过程中可能接触客户Agent的输入输出数据
- 需要签署数据处理协议（DPA）
- 数据不出境（除非满足跨境传输条件）
- 评测数据在评测完成后N天内删除

**GDPR（欧盟）额外要求：**
- DPO（数据保护官）
- DPIA（数据保护影响评估）
- 用户有权要求删除
- 72小时数据泄露通知

**实操建议：**
1. MVP阶段不做欧盟市场，规避GDPR
2. 中国市场的评测数据：本地化存储、评测完删除
3. 合同里写明"龙虾学院不存储客户数据，仅在评测期间临时处理"

### 4. 保险联动的法律框架

**方案：**
- 与保险公司合作推出"Agent责任险"
- 龙虾学院提供评测数据作为核保依据
- 保险公司承保，龙虾学院不承担保险责任
- 类比：体检中心提供体检报告给保险公司，体检中心不赔

**合同关系：**
```
企业 ←→ 保险公司（承保合同）
企业 ←→ 龙虾学院（评测服务合同）
保险公司 ←→ 龙虾学院（数据合作协议）
```

### 5. 5种核心合同模板

| 合同 | 用途 | 关键条款 |
|------|------|---------|
| 评测服务协议 | 客户付费评测 | 免责条款、责任上限、数据处理 |
| 数据处理协议(DPA) | 数据合规 | 处理范围、保留期限、删除义务 |
| 技能包开发者协议 | 技能市场上架 | IP归属、安全审查、收益分成 |
| 合作伙伴协议 | 与BSI/TÜV合作 | 分工、品牌使用、责任分配 |
| 开源许可协议 | 评估框架开源 | Apache 2.0 + 商业许可补充 |
