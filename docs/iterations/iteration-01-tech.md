# 龙虾学院 · 第1轮迭代：工部技术架构深度评审

> 迭代时间：2026-03-23 23:59
> 迭代角度：技术架构可行性

## 核心发现

### 1. Agent指纹方案（P0）

**现状问题：** V2说"config_hash + tool_manifest_hash"但没说怎么防伪造。

**改进方案：**
- 用TPM/TEE做远程证明（attestation），不是简单hash
- Agent启动时生成"运行证明"：包含配置hash + 模型hash + 工具清单 + 环境指纹
- 证明签名用Ed25519，密钥存在TEE/硬件安全模块
- 验证方不信任Agent自己声称的指纹，而是验证TEE签名的attestation report

**类比：** 类似Intel SGX的远程证明——你不信任运行代码的机器，你信任硬件签名的证明。

### 2. 沙箱架构（P0）

**现状问题：** V2说"Docker+seccomp→Firecracker"但没设计安全边界。

**改进方案：**
```
┌─────────────────────────────────────┐
│ Agent运行沙箱（Firecracker microVM）│
│  ┌───────────┐  ┌───────────────┐  │
│  │ Agent进程  │  │ 工具执行器     │  │
│  │ (read-only│  │ (白名单工具)   │  │
│  │  文件系统) │  │ (资源限制)     │  │
│  └───────────┘  └───────────────┘  │
│  网络：仅允许白名单域名              │
│  磁盘：tmpfs，评测结束后销毁         │
│  CPU/Mem：cgroup限制                │
└─────────────────────────────────────┘
        ↑ 评测任务输入（只读挂载）
        ↓ 评测结果输出（签名）
```

**关键设计：**
- 每个评测任务独立microVM（不是共享容器）
- 评测完成后立即销毁，不残留
- 网络隔离：只能访问指定的测试API
- 所有Agent行为被录制为审计日志

### 3. 证书体系（P0）

**方案：**
- 不做X.509（太重），做W3C Verifiable Credentials（VC）
- VC格式：JSON-LD，包含Agent指纹、评测结果、有效期、签发者
- 签名：Ed25519（轻量，适合频繁验证）
- 吊销：用StatusList2021（W3C标准），位图方式，每bit代表一个证书状态
- 验证：任何第三方可以独立验证，不需要连学院服务器

**优势：**
- W3C VC是去中心化身份标准，互操作性好
- 不依赖CA体系
- 可以嵌入到Agent的API响应头中
- 保险/审计方可以离线验证

### 4. 心跳评测（P1）

**方案：**
- Agent定期（每天/每小时）向学院报告运行快照
- 快照包含：最近N次决策的输入输出（脱敏后）、性能指标、错误率
- 学院对快照做抽检（不是全量评测）
- 漂移检测：对比当前行为与基线评测时的行为差异
- 超过阈值触发"复评"

### 5. 技能包权限（P1）

**方案：**
- 每个技能包声明自己需要的权限（capability manifest）
- 权限粒度：工具级别 + 数据级别 + 网络级别
- Agent安装技能包时，学院检查权限是否超出Agent本身的权限边界
- 运行时：技能包不能访问Agent未授权的资源

## 技术选型建议

| 组件 | 建议 | 理由 |
|------|------|------|
| 语言 | Go（CLI）+ Python（评测） | Go编译快、分发简单；Python生态丰富 |
| 沙箱 | Firecracker > gVisor > Docker | 隔离强度递减 |
| 数据库 | PostgreSQL + TimescaleDB | 关系数据+时序监控数据 |
| 消息队列 | NATS（非RabbitMQ） | 更轻量，适合Agent场景 |
| 证书格式 | W3C VC (JSON-LD) | 去中心化标准 |
| 签名算法 | Ed25519 | 快速、轻量 |
| API | gRPC内部 + REST外部 | 内部高效，外部易用 |

## 与V2的差异

| 项目 | V2 | 本轮改进 |
|------|-----|---------|
| 指纹 | config_hash | TEE远程证明 |
| 沙箱 | Docker+seccomp | Firecracker microVM |
| 证书 | "待定义" | W3C VC + StatusList2021 |
| 心跳 | 概念描述 | 漂移检测+抽检机制 |
| 技能包 | 无权限设计 | capability manifest |
