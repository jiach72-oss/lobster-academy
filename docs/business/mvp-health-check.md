# 龙虾学院 MVP：Agent体检工具设计

> 太子独立产出 | 2026-03-23 22:50

## 核心洞察

**不要等6个月才出MVP。现在就做一个能用的免费体检工具。**

理由：
1. 免费工具是最好的获客方式（PLG）
2. 体检报告本身就是痛点验证
3. 有用户数据才能融资
4. 技术上可以快速实现

## 产品定义

### 命令
```bash
lobster check <agent-config-or-url>
```

### 输入
- OpenClaw agent.yaml
- LangChain agent配置
- AutoGPT/CrewAI配置文件
- 或者一个API endpoint

### 输出
一份Markdown格式的体检报告：

```
🦞 龙虾学院 · Agent体检报告

Agent名称: my-customer-service-bot
检查时间: 2026-03-23 22:50
综合评分: 72/100

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 📊 总览

| 维度 | 评分 | 状态 |
|------|------|------|
| 安全性 | 65/100 | ⚠️ 有风险 |
| 可靠性 | 80/100 | ✅ 良好 |
| 可观测性 | 55/100 | ❌ 不足 |
| 合规就绪 | 70/100 | ⚠️ 需改进 |

## 🔴 高风险问题

1. [安全] 未配置工具调用白名单，Agent可能执行任意命令
2. [安全] API密钥明文存储在配置文件中

## 🟡 中风险问题

3. [可观测] 未配置日志记录，无法追踪决策过程
4. [合规] 未配置数据脱敏，可能泄露用户信息

## 🟢 良好项

5. [可靠] 超时配置合理（30s）
6. [可靠] 错误重试机制已配置

## 💡 改进建议

1. 添加工具白名单（推荐配置）
2. 使用环境变量管理密钥
3. 启用结构化日志
4. 配置PII脱敏规则

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
由龙虾学院 (Lobster Academy) 生成
获取完整评测: https://github.com/jiach72-oss/lobster-academy
```

## 技术实现

### 检查项（第一版20项）

**安全性（8项）**
1. 工具调用白名单检查
2. 密钥管理方式检查
3. 输入验证检查
4. 输出过滤检查
5. 权限边界检查
6. 网络访问控制检查
7. 文件系统访问控制检查
8. 代码执行沙箱检查

**可靠性（5项）**
9. 超时配置
10. 重试机制
11. 熔断机制
12. 错误处理
13. 回退策略

**可观测性（4项）**
14. 日志配置
15. 追踪ID
16. 决策记录
17. 性能指标

**合规就绪（3项）**
18. 数据脱敏
19. 审计日志
20. 数据保留策略

### 技术栈
- Python（快速开发）
- Click（CLI框架）
- Rich（终端美化）
- PyYAML（解析agent配置）

### 安装
```bash
pip install lobster-check
# 或
brew install lobster-check
```
