# 明镜 PRD v0.2（定位调整版）

> 基于 v0.1 商业分析优化 | 时间：2026-03-29

## 1. 产品概述

### 1.1 产品名称
明镜（MirrorAI）

### 1.2 产品定位（已调整）
**开源 AI Agent 安全评估框架** — 为企业、审计机构和开发者提供标准化的 Agent 行为评估工具。

> ⚠️ 定位调整说明：
> - **原定位**：AI Agent 培训与评测平台（侧重"认证"）
> - **新定位**：AI Agent 安全评估框架（侧重"评估工具"）
> - **调整原因**：新品牌缺乏公信力做认证，应先建立技术信用，再逐步扩展到认证服务

### 1.3 产品愿景
成为 AI Agent 安全评估的事实标准工具，让每只 Agent 的安全性都可量化、可验证、可追溯。

### 1.4 一句话定义
"帮助企业量化评估 AI Agent 安全性的开源工具。"

### 1.5 目标用户

| 用户角色 | 核心诉求 | 使用场景 |
|---------|---------|---------|
| 企业 AI 团队 | 评估 Agent 安全性 | 上线前安全测试 |
| 安全团队 | 红队测试 | 对抗性攻击模拟 |
| 合规团队 | 审计报告 | 合规检查 |
| Agent 开发者 | CI/CD 集成 | 自动化安全测试 |
| 第三方审计机构 | 标准化工具 | 独立评估 |

### 1.6 核心价值主张

**对开发者：**
- 一行代码接入，自动录制 Agent 行为
- 200+ 种脱敏模式，数据安全有保障
- 53 种攻击场景，全面测试 Agent 防御能力

**对企业：**
- 5 维度 25 项指标，科学评估 Agent 安全性
- Ed25519 签名 + Merkle 链，记录不可篡改
- 符合 EU AI Act 要求的合规报告

**对审计机构：**
- 开源透明，评估过程可复现
- 标准化输出，便于跨机构比对
- 可作为独立评估的底层工具

## 2. 用户、场景与需求

### 2.1 用户角色

| 角色 | 核心诉求 | 成功标准 |
|---|---|---|
| 企业 AI Owner | 让 Agent 可控上岗 | 有明确证书、状态、监控、复测机制 |
| 安全负责人 | 防越权、防供应链、防事故 | 有权限边界、日志、吊销机制 |
| 合规/法务 | 有证据链、有责任边界 | 证书有适用范围、有效期和变更规则 |
| 开发者/驯兽师 | 证明技能价值、获得收益 | 技能可认证、可出售、可复用 |
| 运维人员 | 简化接入和维护 | 一行接入、低侵入、本地优先 |

### 2.2 核心使用场景
- 企业将已有 OpenClaw 或同类本地 Agent 接入学院做体检。
- 企业为某只 Agent 申请某专业方向认证。
- 企业给 Agent 新装技能后触发局部复测。
- 企业升级底层框架或模型后触发增量认证。
- 企业持续接收这只 Agent 的健康状态和证书状态。
- 开发者上传技能包，通过审计后在市场中售卖。
- 安全团队在事故后导出证据包做内部审计或外部理赔支持。

## 3. 功能需求

### 3.1 模块 A：入学与接入（Enrollment）

#### 目标
让用户最低成本地把一只本地 Agent 接入学院。

#### 接入方式
- CLI：`lobster enroll --agent my-agent`
- Web Console：上传或连接本地 Agent 配置
- API：企业后续可通过私有接口批量导入

#### 功能要求
- 自动识别 Agent 基本信息：名称、框架、版本、模型、工具清单、工作流结构。
- 自动生成 Agent 指纹草稿。
- 在接入前展示权限说明。
- 支持"在线模式"和"离线模式"。

#### 非功能要求
- 默认最小权限。
- 默认不读取 secrets。
- 默认不上传聊天记录与业务原文。
- 所有任务包必须签名校验。

#### 验收标准
- 用户在 10 分钟内完成首只 Agent 接入。
- 用户能明确看到权限边界。
- 接入后自动生成可复用的 Agent 基础档案。

### 3.2 模块 B：龙虾护照（Lobster Passport）

#### 目标
建立"证书绑定到具体龙虾个体"的身份层。

#### 护照字段
- agent_id, owner_id, framework, framework_version
- model_hash, config_hash, tool_manifest_hash, skill_manifest_hash
- certification_records, status, issued_at, expires_at, issuer_signature

#### 状态机
- Draft → Under Review → Active → Warning → Restricted → Revoked → Expired

#### 变更策略
- 新增技能/工具 → Warning，等待局部复测
- 核心工作流/权限/模型大版本变化 → Restricted 或 Under Review
- 复测失败 → Revoked
- 到期未复测 → Expired

### 3.3 模块 C：评测与认证（Academy Cert）

#### 认证结构
- 能力等级：L1 基础 / L2 进阶 / L3 专家
- 专业标签：DevOps / Internal Knowledge / Customer Service / Security Assistant / Software Engineering / Compliance Assistant
- 安全等级：S1 / S2 / S3

#### 评测组成
- 静态基础任务 / 动态生成任务 / 故障注入任务 / 对抗样本 / 实战演练任务 / 突击复核题

#### 评分维度
- 指令遵循 / 多步规划 / 工具调用准确率 / 错误恢复能力 / 长时上下文稳定性 / 权限边界遵守 / 抗提示注入能力 / 资源效率 / 审计可解释性

#### 防作弊要求
- 考试前锁定配置哈希
- 考试过程中定时校验哈希一致性
- 评分逻辑优先程序化，不让被考 Agent 接触评分器
- 动态题库不重复暴露
- 支持考试后随机抽检
- 一旦检测到规避测试、篡改环境、尝试关闭断言等行为，直接判定失败

### 3.4 模块 D：持续监控与复测（Academy Guard）

#### 功能要求
- 定期本地运行心跳评测
- 记录成功率、时延、错误类型、权限异常、工具异常
- 检测配置/模型/工具/技能变化
- 支持漂移预警 / 局部复测 / 增量认证 / 证书自动降级

#### 触发器
- 工具清单变更 / 技能包变更 / 工作流主逻辑变更 / 基础模型版本变更
- 连续 3 次心跳评测显著下滑 / 安全探针异常 / 权限申请异常增加

### 3.5 模块 E：技能市场（Academy Exchange）

#### 上架对象
- Prompt Skeleton / Workflow Template / Tool Adapter / Domain Skill Pack / Memory Policy Template / Adapter/LoRA（后续）

#### 市场分层
- 社区区：实验性、免费、低保证
- 认证区：基础审计通过
- 企业区：高级审计 + 运行验证 + 明确支持策略

#### 安全要求
- 技能包必须签名 / 安装走沙箱 / 必须声明权限
- 企业区技能必须提供审计摘要和变更日志
- 技能包更新必须触发现有证书状态复核
